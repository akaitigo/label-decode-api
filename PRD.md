# PRD: label-decode-api

**作成者**: Ryusei | **日付**: 2026-03-29 | **ステータス**: ドラフト
**アイデアID**: #793 | **ドメイン**: 食品添加物/成分表示 | **難易度**: Medium

## 問題定義

食品パッケージの一括表示（原材料名・添加物・アレルゲン・栄養成分表示）はテキスト形式で記載されており、プログラムから構造化データとして扱うことが困難である。特に日本の食品表示法（2020年改正）では「スラッシュルール」により添加物と原材料の境界がスラッシュ（/）で区切られるが、この解析には専門知識が必要。小売・食品メーカー・ヘルスケアアプリが個別にパーサーを実装しており、重複開発と品質のばらつきが発生している。

## 目標と成功指標

| 目標 | 指標 | ターゲット |
|------|------|----------|
| 食品表示テキストの構造化精度 | 原材料・添加物の正規化成功率 | 95%以上（一般的な加工食品） |
| アレルゲン検出の網羅性 | 特定原材料等28品目の検出率 | 100%（表示義務7品目）、95%（推奨21品目） |
| API応答速度 | 95パーセンタイルレイテンシ | 200ms以下 |
| 開発者の導入容易性 | クイックスタートからAPI呼び出しまでの時間 | 5分以内 |
| 添加物分類の正確性 | 用途別分類の一致率 | 90%以上 |

## スコープ外

- OCR（画像からのテキスト抽出）— 入力はテキスト前提
- 栄養計算・カロリー計算 — 成分の推定のみ
- 多言語対応 — MVP は日本語のみ
- ユーザー認証・認可 — MVP ではオープンAPI
- 食品安全性の医学的判断 — 分類のみ、安全性の保証はしない

## ユーザーストーリー

- 食品アプリ開発者として、一括表示テキストを送信して構造化JSONを取得したい。なぜならアプリ内で原材料をフィルタリング表示するため
- アレルギー管理アプリ開発者として、食品テキストからアレルゲンを自動検出したい。なぜなら手動入力はユーザー負担が大きくエラーが発生するため
- 食品メーカーの品質管理担当として、自社製品の表示テキストが法令準拠かチェックしたい。なぜなら添加物の用途表記漏れは法的リスクがあるため
- 小売バイヤーとして、仕入れ候補商品の添加物プロファイルを一括比較したい。なぜなら「無添加」訴求商品のバリデーションを効率化するため

## 技術要件

### 技術スタック

- **サーバー**: Kotlin / Quarkus（gRPC Server）
- **NLPパーサー**: Python（食品表示テキスト解析）
- **データベース**: PostgreSQL（添加物マスタ・分類辞書）
- **インフラ**: GCP Cloud Run
- **Proto管理**: buf

### アーキテクチャ

Kotlin/Quarkus gRPCサーバーがフロントエンドAPIを提供し、Python NLPパーサーをサブプロセスまたはgRPCで呼び出す構成。MVPではPythonパーサーをKotlinプロセス内からコマンドライン呼び出しし、v2でgRPCサービス間通信に移行する。

### 非機能要件

- パフォーマンス: p95 200ms以下（単一テキスト解析）
- セキュリティ: 入力テキストのサニタイズ、SQLインジェクション防止
- スケーラビリティ: Cloud Run オートスケール（同時接続100）

## 競合分析

| 競合 | 概要 | ギャップ |
|------|------|---------|
| FatSecret API | 栄養情報DB API | 日本の食品表示法非対応、テキスト解析なし |
| Edamam API | 食品・栄養解析API | 英語中心、日本の添加物分類未対応 |
| Open Food Facts | オープンソース食品DB | 日本語対応不十分、リアルタイム解析API なし |

**差別化ポイント**: 日本特有の「スラッシュルール」（2020年改正食品表示法）に対応した添加物抽出パーサーと、業務用途向けgRPC APIの提供。

## 技術リスク

| リスク | 発生可能性 | 影響 | 軽減策 |
|--------|----------|------|--------|
| 日本語NLPの精度不足 | 中 | 高 | ルールベース+辞書ベースのハイブリッド。ML は v2 |
| 添加物マスタの網羅性 | 中 | 中 | 厚労省公開リストをベースに段階的拡充 |
| Kotlin-Python 間連携の複雑性 | 低 | 中 | MVP はプロセス呼び出し、v2 で gRPC 化 |
| 食品表示法の改正追従 | 低 | 高 | マスタデータを DB 管理し、コード変更なしで更新可能にする |
